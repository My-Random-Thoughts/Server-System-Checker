WMI Query

WMI Queries allow you to create custom queries to interrogate your servers hardware and software information.

Instructions

To start adding a WMI Query resources to your group structure,

  1. Select the group or sub-group that you want to add resources too,
  2. Right-click either:
    1. The group in the left-hand group window, or,
    2. Right-click the right-hand resource windows,
  3. Choose   Add Resource from the pop-up menu,
  4. Choose   WMI Query from the pop-out menu,
  5. From the new window that appears, choose a server to connect to from the drop down list and click Connect,
  6. A filtered list of WMI classes will be retrieved and shown, choose the one you want to work with.


The WMI Query Builder will be enabled and you can start building your query...

  1. Click the button in the SELECT window,
  2. A list of class specific properties will be listed. Select one to add to the query,
  3. Continue adding properties as needed.
  4. If you want to filter the results returned, double-click on Add WHERE Clause,
  5. From the window that appears, fill in the details required...
    1. Property : The list of properties added to the SELECT field,
    2. Operator : The compare operation to perform (equals, does not equal, less than, greater than,
    3. Value : The value to compare with.
  6. Click OK to add the Where clause,
  7. Continue adding clauses as needed.
  8. If required, change the AND/OR option for all the rules.
  9. If more than one result is returned, you will need to tick   Return Count Only
  10. Click Test Query to check the query executes successfully,
  11. Select the compare operation to check the result against.
  12. Click OK to add the entered resources to the selected group.

You can now continue to add new resources. Remember, resources are hierarchical.

Please Note

Query results must produce either a TRUE or FALSE answer, therefore any query that returns a list of items (services, processes, etc) must be set as a list count comparison.
For example: SELECT Name FROM Win32_Process would produce a long list of processes currently running on a server.
Filtering this list with WHERE clauses will reduce this list into a meaningful result, from which you can check either...

  • against one specific process - does process "abc" exist, or,
  • a count of specific processes - does the number of processes equal "x"
