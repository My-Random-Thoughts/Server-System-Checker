Getting Started - A Quick Start Guide

When you create a new configuration profile, you are given the option to start with an example layout. Choosing No to this will give you a blank window. From here you will want to start creating some logical groups for your scanning needs.

Groups can represent anything you need them to...

  1. Functions: Domain Controllers, Terminal Servers, Development Servers.
  2. Locations: London, Tokyo, New York.
  3. Importance: Production, Testing, Development

Adding Groups

Groups are used for organising your servers into a logical and structured way. Groups help to organise servers into units so that different resource checks can be set against different groups

By default, a groups colour is yellow, however there are several different colours to choose from when creating or editing a group. See the section titled Group Properties for more information.

To start creating your groups,

  1. Right-click anywhere inside the left hand panel,
  2. Choose   Add Group from the pop-up menu,
  3. Enter a name for this new group.

You can now continue to add new groups, or sub-groups until your structure is complete.

Adding Servers

Servers are the remote machines that we want to scan.

They can be added by either

  1. typing one at a time,
  2. importing from a text file, or
  3. scanning the current active directory domain

To start adding servers to your group structure,

  1. Select and right-click the group that you want to add servers too,
  2. Choose   Add Server(s) from the pop-up menu,
  3. From the new window that appears, either:
    1. Click the
      2. Add button and enter a server name,
    2. From the Import From : drop down list, choose:
      1. Active Directory
      2. External Text File
  4. Click OK to add the servers to the selected group.

Note: If you add an IP address, the application will try to resolve it into a name using DNS, if the   Resolve All IP addresses is selected.

Adding Resources

Resources are added to group levels, and take a hierarchical nature. That is, a resource added to the root of a large sub-group tree, will be scanned at every level of that tree.

To start adding resources to your group structure,

  1. Select the group or sub-group that you want to add resources too,
  2. Right-click either:
    1. The group in the left-hand group window, or,
    2. Right-click the right-hand resource windows,
  3. Choose   Add Resource from the pop-up menu,
  4. Choose one of the resource types from the pop-out menu
    1.   Eventlog Scan or,
    2.   File Check or,
    3.   Freespace Threshold or,
    4.   Hotfix Patch Check or,
    5.   Registry Scan or,
    6.   WMI Query or,
    7.   Windows Services.
  5. From the new window that appears complete the required details
    (detailed information is shown later).
  6. Click OK to add the selected resource to the selected group.

You can now continue to add new resources. Remember, resources are hierarchical.

Scanning The Selected Group

Once you have your environment configured as you need it, you can start to scan your groups. Scanning before a large task will allow you to get a base line of results to compare against.

To start scanning a group,

  1. Select and right-click the group you want to scan,
  2. Choose   Scan Group from the pop-up menu,

A new scan window will appear, and will show the tree structure of the selected group, along with all the servers in that structure. The scan will start automatically, and a progress bar will show its current status.

Each server will scanned in turn, with the current status of each resource type shown.

Once the scan has completed, you can hover over each scan result icon to see some brief information about the result. From here you have several options:

  1. Double-click a server to see it's detailed scan report, or,
  2. Click the Export button to produce a summary of the results currently shown.
  3. If a server was rejected or not scanned, you can right-click it and select   Re-scan Selected Server

Detailed Scan Report

The detailed scan report will give you the complete picture of any errors or warnings that may have occurred when scanning the selected server. This window has several tabs along the top, one for each scan area.

Windows Services

This tab shows any windows services that are not as expected. That is, any stopped service that should be started, and any started service that should be stopped.

Hotfix Patches

This tab shows any hotfixes that are not as expected. That is, any missing hotfixes that should be installed, and any installed hotfixes that shouldn't be there.

Eventlog Scan

This tab shows any eventlog entries that have appeared since the selected server was last rebooted.

Registry Scan

This tab shows any registry entries that are incorrect or missing.

File Check

This tab shows the results of the file scanning check.

WMI Query

This tab shows the results of the custom WMI Queries that you have configured

Freespace Threshold

This tab shows the selected servers main system drive (usually C:) as a pie chart representing the free space available. It also lists any other local drives to this server, and the free space available.

Export Result Summary

This option allows you to export the current scan results to a HTML based report for saving and viewing at a later date.

To create a new HTML based report,

  1. Select either:
    1.   Simple (default), or,
    2.   Detailed


    3.   Include Image Icons (increases report file size)
    4.   Include Server Uptime (if available)


      5. If you chose the detailed report, you have the following further options:
      1.   List all incorrect services
      2.   List all incorrect hotfixes
      3.   List every reported eventlog entry
      4.   List all missing registry keys
      5.   List every incorrect file
      6.   List all WMI query results
      7.   Show system drive space status
      8.      List all scanned drives
      9.         Add excluded drives
  2. Click OK to run the report and save it to the current folder.

Including image icons into the report will increase the size of the resulting report file, but will give a nicer looking and more professional output.
